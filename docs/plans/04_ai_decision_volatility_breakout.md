# 4. AI 판단 리뷰 (변동성돌파 관점, 상용화 목적)

작성일: 2026-01-02

---

## 1. 현재 AI 사용 요약
- 1H 진입 판단에서 AI 사용
- JSON 응답 강제
- 보수적 프레이밍(HOLD 허용)

---

## 2. 장점

- 돌파 전략에 ‘진입 억제 필터’로 적합
- AI를 관리(15m)에 쓰지 않음 → 안정적
- 기술지표 요약 후 판단 구조

---

## 3. 상용 리스크

### (1) 프롬프트 과책임
- AI가 방향성/신뢰도 직접 판단
- 결정론/재현성 약함

### (2) 파라미터 통제 미흡
- temperature / timeout / retry 일관성 부족

### (3) 입력 데이터 과다
- 원시 데이터 → 토큰 비용/판단 오류 증가

---

## 4. 리팩토링안 (프롬프팅 전격 리뷰)

### (1) AI 역할 축소
- AI = 돌파 “허용/차단” 필터
- 매수 가격/수량/스탑은 규칙 기반

### (2) 입력 표준화
- raw OHLC ❌
- regime/ATR/돌파강도/리스크예산 요약값만 제공

### (3) 결정 추적성
- prompt_version, model, params, decision, PnL 라벨 DB 저장

### (4) 보안/안정성
- JSON schema validation
- 실패 시 HOLD
- 호출 rate limit / circuit breaker

---

## 5. 결론
AI는 변동성돌파의 **엔진이 아니라 브레이크**여야 한다.
상용 목적이면 “덜 똑똑하지만 일관된 AI”가 더 돈을 지킨다.
